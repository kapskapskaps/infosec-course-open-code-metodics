# Урок 7. Вредоносное ПО и защита

## Методика

**Цель урока:** Научить ученика распознавать, анализировать и защищаться от вредоносного ПО.

**Задачи:**
- Классифицировать типы вредоносного ПО (вирусы, трояны, шифровальщики, шпионы, майнеры)
- Разобрать жизненный цикл заражения
- Провести анализ подозрительных файлов в изолированной среде
- Научить основам безопасного поведения при подозрении на заражение
- Настроить базовые средства защиты

**Тип урока:** Индивидуальный онлайн

**Оборудование:** Компьютер, Windows Sandbox (или VirtualBox), Process Explorer, Autoruns, VirusTotal, браузер

**Критерии успеха:**
- Ученик классифицирует 5+ типов вредоносного ПО
- Ученик анализирует подозрительный файл через VirusTotal
- Ученик проверяет систему на признаки заражения
- Ученик создаёт план действий при подозрении на вирус

## 1. Теория (15 минут)

### 1.1. Что такое вредоносное ПО (Malware)

**Malware** (Malicious Software) — любое программное обеспечение, созданное для нанесения ущерба.

### 1.2. Классификация (7 основных типов)

| Тип | Описание | Пример |
|-----|----------|--------|
| **Вирус** | Внедряется в другие программы, саморазмножается | CIH (Чернобыль), ILOVEYOU |
| **Червь (Worm)** | Самостоятельно распространяется по сети, не требует внедрения | Morris worm, Stuxnet, Mydoom |
| **Троян (Trojan)** | Маскируется под полезную программу | WinLocker, Zeus |
| **Шифровальщик (Ransomware)** | Шифрует файлы, требует выкуп | WannaCry, NotPetya |
| **Шпион (Spyware)** | Собирает данные о пользователе | Pegasus, FinFisher |
| **Майнер (Cryptominer)** | Использует компьютер для майнинга без ведома | Coinhive |
| **Руткит (Rootkit)** | Скрывает своё присутствие в системе | Sony BMG rootkit |

### 1.3. Жизненный цикл заражения

1. **Вектор атаки** — как вирус попал на компьютер (фишинг, заражённый USB, уязвимость, пиратский софт)
2. **Запуск** — исполнение вредоносного кода (двойной клик, открытие документа, автозагрузка)
3. **Закрепление (Persistence)** — вирус обеспечивает себе запуск при каждой загрузке (реестр, автозагрузка, планировщик задач)
4. **C2 (Command & Control)** — соединение с сервером управления для получения команд
5. **Вредоносное действие** — кража данных, шифрование, майнинг, рассылка спама
6. **Сокрытие** — маскировка, обход антивирусов, шифрование своего кода

### 1.4. Пути заражения

**Статистика (2024-2025):**
- 65% заражений — через фишинг
- 15% — через заражённые сайты (drive-by download)
- 10% — через пиратское ПО и кейгены
- 5% — через съёмные носители (USB)
- 5% — через уязвимости в ПО

### 1.5. Наиболее опасные кампании в истории

**ILOVEYOU (2000)**
- Червя, который маскировался под письмо "LOVE-LETTER-FOR-YOU.txt.vbs"
- Уничтожал файлы (фото, музыку) и рассылал себя всем контактам
- Ущерб: 8.7 млрд долларов (с поправкой на инфляцию)

**Stuxnet (2010)**
- Червя, созданный США и Израилем для атаки на иранские ядерные центрифуги
- Портил оборудование, показывая ложные показатели датчиков
- Первое в истории кибероружие

**WannaCry (2017)**
- Шифровальщик, заразивший 300 000 компьютеров в 150 странах
- Использовал уязвимость EternalBlue (разработана АНБ США)
- Ущерб: 4+ млрд долларов
- Был остановлен случайно — исследователь безопасности зарегистрировал домен, к которому обращался вирус

### 1.6. Признаки заражения

**Чек-лист "Вирус или нет":**
1. Компьютор стал заметно медленнее работать
2. Постоянная активность жёсткого диска/SSD (индикатор постоянно горит)
3. Всплывающие окна с рекламой
4. Перенаправление на незнакомые сайты в браузере
5. Файлы не открываются (изменились иконки, расширения .encrypted, .locked)
6. Антивирус отключён и не включается
7. Программы самопроизвольно запускаются и закрываются
8. Высокая загрузка CPU/GPU в простое (возможно — майнер)
9. Подозрительные процессы в диспетчере задач
10. Сетевой трафик без видимой причины

## 2. Практика (60 минут)

### Задание 1. VirusTotal — анализ подозрительных файлов (10 минут)

**Инструкция:** Ученик заходит на https://www.virustotal.com и анализирует подозрительные файлы.

**Шаг 1. Анализ хэша файла:**
Преподаватель даёт хэши известных файлов (документы, exe-файлы).

```python
# Сначала вычисляем SHA-256 для подозрительного файла
import hashlib
hash_file = hashlib.sha256(open("подозрительный.exe", "rb").read()).hexdigest()
print(hash_file)
```

1. Загрузить хэш на VirusTotal
2. Посмотреть: сколько антивирусов определяют файл как вредоносный?
3. Какие имена угроз?
4. Какое поведение файла (network connections, registry changes)?

**Шаг 2. Анализ ссылки:**
1. Открыть VirusTotal → URL
2. Вставить подозрительную ссылку
3. Посмотреть: какие категории? (phishing, malware, malicious)

**Шаг 3. Анализ по IP:**
1. Вставить IP-адрес на VirusTotal
2. Какие домены привязаны?
3. Есть ли в чёрных списках?

**Вопросы ученику:**
1. Всегда ли 1/60 детектов означает, что файл безопасен? (нет — может быть свежий или редкий)
2. Всегда ли 60/60 означает, что файл опасен? (обычно да, но бывают ложные срабатывания)
3. Какой порог для вас лично: сколько антивирусов должно определить файл как опасный?

### Задание 2. Анализ процессов в диспетчере задач (10 минут)

**Инструкция:** Ученик открывает Диспетчер задач (Ctrl+Shift+Esc) и проводит аудит запущенных процессов.

**Шаг 1. Анализ процессов:**
Ученик должен найти и записать:
- Названия всех подозрительных процессов
- Потребление CPU и RAM каждым процессом
- Производитель (Microsoft / сторонний / неизвестный)

**Шаг 2. Проверка подозрительных процессов:**
1. Правой кнопкой по процессу → Открыть расположение файла
2. Проверить: легитимный ли это путь? (например, `C:\Windows\System32\svchost.exe` — нормально, `C:\Users\...\svchost.exe` — подозрительно)
3. Правой кнопкой → Свойства → Цифровая подпись (есть ли?)

**Шаг 3. Process Explorer (продвинутый диспетчер):**
Скачать Process Explorer от Sysinternals (Microsoft).

*Запустить и найти:*
1. Какие процессы имеют активные сетевые соединения?
2. Какие процессы загружены из необычных путей?
3. Какие процессы без цифровой подписи?
4. Какие процессы выделены красным/розовым? (Process Explorer окрашивает подозрительные)

**Чек-лист для проверки процессов:**

| Процесс | Легитимный? | Путь | CPU | RAM | Цифровая подпись | Сеть |
|---------|------------|------|-----|-----|-----------------|------|
| svchost.exe | Да | C:\Windows\System32 | 2% | 50 MB | Microsoft | Нет |
| explorer.exe | Да | C:\Windows | 1% | 80 MB | Microsoft | Нет |
| chrome.exe | Да | C:\Program Files\Google | 15% | 500 MB | Google | Да |
| ... | ... | ... | ... | ... | ... | ... |

**Критерии:** 5+ подозрительных кандидатов → проверить детально.

### Задание 3. Проверка автозагрузки (10 минут)

**Инструкция:** Ученик проверяет, какие программы запускаются при старте Windows.

**Шаг 1. Через Диспетчер задач:**
1. Ctrl+Shift+Esc → Автозагрузка
2. Оценить каждую программу: что это? откуда? нужна ли?
3. Отключить подозрительные

**Шаг 2. Autoruns (Sysinternals):**
Скачать и запустить Autoruns.

*Найти:*
1. Записи в реестре (HKCU\Software\Microsoft\Windows\CurrentVersion\Run)
2. Запланированные задачи (Scheduled Tasks)
3. Расширения браузера (Browser Extensions)
4. Службы (Services)

*Отфильтровать:*
- Скрыть подписанные Microsoft → останутся только сторонние
- Просмотреть каждую на подозрительность

**Шаг 3. Создание "белого списка":**
Ученик должен создать список ТОЛЬКО необходимых программ для автозагрузки.

### Задание 4. Анализ в изолированной среде (Sandbox) (15 минут)

**Инструкция:** Ученик запускает подозрительный файл в изолированной среде (Windows Sandbox или VirtualBox).

**Вариант A: Windows Sandbox (если Windows 10/11 Pro)**

1. Открыть Windows Sandbox
2. Скопировать подозрительный файл в Sandbox
3. Запустить
4. Наблюдать через Process Explorer внутри Sandbox
5. Закрыть Sandbox — все изменения отброшены

**Вариант B: Без песочницы — теоретический анализ**

Преподаватель даёт лог поведения файла (из реального анализа):

```
Файл: setup_activator.exe
Хэш SHA-256: f7a9e3b1c2d5...
Детекция: 32/68 антивирусов

Сетевые соединения:
- 185.220.101.42:8443 (C2-сервер, Германия)
- api.telegram.org (кража данных через Telegram)

Изменения реестра:
- HKCU\...\Run → added: "WindowsUpdater" = "%TEMP%\svchost.exe"
- HKLM\...\Winlogon → modified

Файловые операции:
- %TEMP%\svchost.exe (235 KB)
- C:\Users\...\Documents\*.docx → зашифрованы

Признаки:
- Persistence: автозагрузка через реестр
- C2: соединение с неизвестным сервером
- Ransomware: шифрует документы
```

**Ученик должен:**
1. Классифицировать вредоносное ПО (троян? шифровальщик? майнер?)
2. Определить: как попал? как закрепился? какой ущерб?
3. Составить план действий

### Задание 5. План действий при заражении (15 минут)

**Инструкция:** Ученик разрабатывает пошаговый план действий при подозрении на заражение.

**Ученик пишет документ:**

```
# ПЛАН ДЕЙСТВИЙ ПРИ ЗАРАЖЕНИИ ВИРУСОМ

## Шаг 1. НЕ ПАНИКОВАТЬ
- Отключить компьютер от интернета (Wi-Fi, кабель)
- Отключить Bluetooth

## Шаг 2. ОЦЕНКА УЩЕРБА
- Работают ли файлы? Есть ли признаки шифрования?
- Какие данные могли быть скомпрометированы? (пароли, документы, фото)
- Это шифровальщик (ransomware)? Есть ли записка с требованием?

## Шаг 3. ТИП ЗАРАЖЕНИЯ
- Проверить VirusTotal (если есть доступ)
- Проверить диспетчер задач и автозагрузку
- Есть ли подозрительные процессы?

## Шаг 4. ДЕЙСТВИЯ (в порядке возрастания серьёзности)
1. Запустить полное сканирование антивирусом (Windows Defender, Kaspersky)
2. Загрузиться в безопасном режиме (F8 при загрузке)
3. Запустить AdwCleaner или Malwarebytes (бесплатные)
4. Проверить автозагрузку через Autoruns
5. Если шифровальщик → обратиться к специалисту. НЕ платить выкуп!
6. Если ничего не помогает → переустановка Windows

## Шаг 5. ПОСЛЕ ЛЕЧЕНИЯ
- Сменить ВСЕ пароли (особенно почту, соцсети, банки)
- Подключить 2FA везде
- Сделать резервную копию
- Выяснить, как произошло заражение (чтобы не повторить)
```

**Важно:** Никогда не платить выкуп шифровальщикам! Есть 3 причины:
1. Нет гарантии, что файлы расшифруют
2. Вы финансируете преступников
3. Вас могут внести в "список плательщиков" и атаковать снова

**Дополнительный блок: безопасное использование антивирусов**

1. Windows Defender (встроенный) — хороший базовый уровень
2. Не ставить 2 антивируса одновременно (конфликт, замедление)
3. Регулярное полное сканирование (раз в месяц)
4. Важно: антивирус не защищает от социальной инженерии и человеческой глупости
5. Крайне полезно: Malwarebytes как дополнительный сканер (по требованию)

## Дополнительно для преподавателя

- Не давать ученику реальные вредоносные файлы! Использовать тестовые сигнатуры (EICAR) или хэши известных образцов
- EICAR test file: `X5O!P%@AP[4\PZX54(P^)7CC)7}$EICAR-STANDARD-ANTIVIRUS-TEST-FILE!$H+H*`
- Подчеркнуть этический аспект: создание и распространение вирусов — уголовно наказуемо (ст. 273 УК РФ)
- Если ученик интересуется защитой глубже: посоветовать курсы по обратной разработке (но с 18 лет)
- Stuxnet — хороший пример для дискуссии о кибервойне
