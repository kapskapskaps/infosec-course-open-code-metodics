## 1. Теория (15 минут)

### 1.1. Что такое криптография?

**Криптография** — наука о методах обеспечения конфиденциальности и аутентификации информации.

**Основные понятия:**
- **Открытый текст (plaintext)** — исходное сообщение
- **Шифротекст (ciphertext)** — зашифрованное сообщение
- **Ключ (key)** — секретная информация для шифрования/расшифровки
- **Шифр (cipher)** — алгоритм шифрования

**Правило Керкгоффса (1883):** "Безопасность шифра должна основываться только на секретности ключа, а не на секретности алгоритма."

То есть: пусть весь мир знает, КАК ты шифруешь — но без ключа никто не сможет расшифровать.

### 1.2. Исторические шифры

**Шифр Цезаря (I век до н.э.)**
- Сдвиг каждой буквы на фиксированное число позиций
- Ключ: число сдвига (например, 3)
- Пример: сдвиг 3: A→D, B→E, C→F... "HELLO" → "KHOOR"
- **Стойкость:** 26 вариантов (взламывается за 1 секунду)

**Шифр Виженера (XVI век)**
- Использует ключевое слово для сдвига каждой буквы
- Каждая буква сдвигается на разное число (по ключу)
- **Стойкость:** Гораздо выше Цезаря (до XIX века считался невзламываемым)
- **Уязвимость:** Частотный анализ + угадывание длины ключа

**Энигма (Вторая мировая война)**
- Роторная шифровальная машина
- Ключ: начальная позиция роторов и коммутация
- Немцы считали абсолютно надёжной
- **Взломана:** Аланом Тьюрингом и польскими математиками

**Важный урок истории:** Никогда не говори "это невозможно взломать".

### 1.3. Симметричное шифрование

**Как работает:**
```
Отправитель: Открытый текст + Ключ → Шифротекст
Получатель:   Шифротекст + Ключ → Открытый текст
```

**Современные алгоритмы:**
- **AES (Advanced Encryption Standard)** — самый популярный симметричный шифр
- **ChaCha20** — современный, быстрый, используется в TLS

**Проблема:** Нужно безопасно передать ключ получателю. Если ключ перехватят — шифрование бесполезно.

### 1.4. Асимметричное шифрование

**Как работает:**
- Есть два ключа: **открытый (public)** и **закрытый (private)**
- Открытым ключом можно зашифровать, но НЕ расшифровать
- Расшифровать может только закрытый ключ

**Схема:**
```
Алиса хочет отправить сообщение Бобу:
1. Алиса берёт открытый ключ Боба (он доступен всем)
2. Алиса шифрует сообщение открытым ключом Боба
3. Сообщение может расшифровать ТОЛЬКО закрытый ключ Боба
4. Даже тот, кто перехватил сообщение, не может его прочитать
```

**Примеры алгоритмов:**
- **RSA** — основан на сложности разложения больших чисел на множители
- **ECC (Elliptic Curve Cryptography)** — современнее, быстрее RSA
- **Диффи-Хеллман (Diffie-Hellman)** — протокол обмена ключами

### 1.5. Хэш-функции

**Хэш** — это "отпечаток" данных фиксированной длины.

**Свойства:**
1. **Детерминированность:** Одни и те же данные → один и тот же хэш
2. **Необратимость:** По хэшу нельзя восстановить исходные данные
3. **Устойчивость к коллизиям:** Крайне сложно найти два разных сообщения с одинаковым хэшем

**Применение:**
- Хранение паролей (в базе хранится хэш, а не пароль)
- Проверка целостности файлов (совпадает ли хэш скачанного файла)
- Цифровые подписи

**Примеры алгоритмов:**
- MD5 — устарел, небезопасен (коллизии найдены)
- SHA-256 — безопасен, используется в TLS, блокчейне

### 1.6. Как работает HTTPS (TLS Handshake)

1. **Приветствие:** Браузер говорит серверу, какие протоколы поддерживает
2. **Сертификат:** Сервер отправляет свой сертификат с открытым ключом (RSA или ECC)
3. **Проверка:** Браузер проверяет, что сертификат подписан доверенным центром (CA)
4. **Обмен ключами:** Клиент и сервер договариваются о сессионном ключе (асимметричное шифрование)
5. **Сессия:** Все данные шифруются сессионным ключом (симметричное шифрование — AES/ChaCha20)

### Задание 2. Программирование шифра Цезаря на Python (15 минут)

**Инструкция:** Ученик пишет программу для шифрования/расшифровки текста.

**Шаг 1. Базовая реализация:**

```python
def caesar_encrypt(text, shift):
    result = ""
    for char in text.upper():
        if char.isalpha():  # Только буквы
            # Получаем номер буквы (0-32 для русского алфавита)
            # Используем Unicode-коды для русского алфавита
            code = ord(char)
            # Русские буквы: А=1040, Я=1071
            if 1040 <= code <= 1071:
                shifted = (code - 1040 + shift) % 33 + 1040
                result += chr(shifted)
            elif 1072 <= code <= 1103:  # строчные
                shifted = (code - 1072 + shift) % 33 + 1072
                result += chr(shifted)
            else:
                result += char
        else:
            result += char
    return result

def caesar_decrypt(text, shift):
    return caesar_encrypt(text, -shift)
```

**Шаг 2. Перебор всех ключей (Brute Force):**

```python
def brute_force(text):
    for shift in range(33):
        decrypted = caesar_decrypt(text, shift)
        print(f"Сдвиг {shift}: {decrypted}")

# Тест
encrypted = caesar_encrypt("Привет, мир!", 10)
print(f"Зашифровано: {encrypted}")
brute_force(encrypted)
```

**Шаг 3. Автоматический взлом (частотный анализ):**

```python
import string

def frequency_analysis(text):
    freq = {}
    total = 0
    for char in text.upper():
        if 1040 <= ord(char) <= 1071:
            freq[char] = freq.get(char, 0) + 1
            total += 1
    
    sorted_freq = sorted(freq.items(), key=lambda x: x[1], reverse=True)
    
    # Самая частая буква в русском языке — О (код 1054)
    if sorted_freq:
        most_common = sorted_freq[0][0]
        # Предполагаем, что most_common = О
        shift = (ord(most_common) - 1040 - 14) % 33  # О = 14-я буква (0-based)
        return -shift  # минус — для расшифровки
    return 0

# Тест
encrypted = caesar_encrypt("Криптография это увлекательно", 15)
print(f"Зашифровано: {encrypted}")
shift = frequency_analysis(encrypted)
print(f"Предполагаемый сдвиг: {shift}")
print(f"Расшифровано: {caesar_decrypt(encrypted, shift)}")
```

### Задание 3. Хэширование на практике (10 минут)

**Инструкция:** Ученик работает с хэш-функциями через Python.

**Шаг 1. Вычисление хэшей:**

```python
import hashlib

text = "Привет, мир!"
print(f"MD5:    {hashlib.md5(text.encode()).hexdigest()}")
print(f"SHA-1:  {hashlib.sha1(text.encode()).hexdigest()}")
print(f"SHA-256: {hashlib.sha256(text.encode()).hexdigest()}")

# Свойства хэша:
# 1. Разная длина входа — одинаковая длина выхода
print(f"\nSHA-256 от 'a':    {hashlib.sha256(b'a').hexdigest()}")
print(f"SHA-256 от 'abcd': {hashlib.sha256(b'abcd').hexdigest()}")
# Оба — 64 символа (256 бит)
```

**Шаг 2. Лавинный эффект:**

```python
text1 = "Криптография — это интересно"
text2 = "Криптография — это интересно!"

print(f"SHA-256 1: {hashlib.sha256(text1.encode()).hexdigest()}")
print(f"SHA-256 2: {hashlib.sha256(text2.encode()).hexdigest()}")
# Добавили "!" в конце — хэш изменился полностью!
```

**Шаг 3. Проверка целостности файла:**

```python
def hash_file(filename):
    with open(filename, "rb") as f:
        return hashlib.sha256(f.read()).hexdigest()

# Создадим файл
with open("test.txt", "w") as f:
    f.write("Это важный документ!")

hash1 = hash_file("test.txt")
print(f"Хэш файла: {hash1}")

# Изменим файл
with open("test.txt", "a") as f:
    f.write(".")

hash2 = hash_file("test.txt")
print(f"Хэш после изменения: {hash2}")
print(f"Совпадают: {hash1 == hash2}")  # False
```

### Задание 4. Взлом MD5-хэша (учебный) (10 минут)

**Инструкция:** Ученик получает MD5-хэш простого пароля и должен подобрать исходный текст.

```python
import hashlib
import string
import itertools

# Целевой хэш (пароль из 3-4 символов)
target_hash = hashlib.md5("xyz".encode()).hexdigest()
print(f"Взламываем хэш: {target_hash}")

chars = string.ascii_lowercase  # 26 букв
found = False

# Сначала 3 символа
for length in [3, 4]:
    if found:
        break
    print(f"Перебираем {length} символа...")
    for combo in itertools.product(chars, repeat=length):
        password = "".join(combo)
        if hashlib.md5(password.encode()).hexdigest() == target_hash:
            print(f"[+] Пароль найден: {password}")
            found = True
            break
```

3. Почему MD5 считается устаревшим? (коллизии найдены, слишком быстрый)
