## 1. Теория (15 минут)

### 1.1. Анатомия пароля: почему "123456" — это катастрофа?

**Статистика 2024–2025:**
- Самый популярный пароль в мире: "123456" — его используют 4,5 млн человек
- Второй по популярности: "password"
- Пароль "qwerty123" взламывается за 0.0003 секунды
- Пароль "jK8#mP2$vL9@xR5!" — 3 × 10^16 лет при переборе 10 млрд паролей/сек

**Почему люди выбирают слабые пароли?**
- Лень
- "Меня никто не взломает" — ложное чувство безопасности
- Слишком много аккаунтов — перегрузка паролями
- Непонимание механизмов взлома

### 1.2. Четыре метода взлома паролей

**Метод 1: Brute Force (Полный перебор)**

Перебираются ВСЕ возможные комбинации символов.

| Длина пароля | Только цифры (10) | Маленькие буквы (26) | Все символы (95) |
|-------------|-------------------|---------------------|------------------|
| 4 символа | 10 000 (мгновенно) | 456 976 (мгновенно) | 81 млн (секунды) |
| 6 символов | 1 млн (секунды) | 308 млн (минуты) | 735 млрд (дни) |
| 8 символов | 100 млн (часы) | 208 млрд (месяцы) | 6.6 × 10^15 (годы) |
| 12 символов | 10^12 (годы) | 9 × 10^16 (века) | 5.4 × 10^23 (космология) |

**Формула:** `Количество комбинаций = (алфавит)^длина`

**Метод 2: Dictionary Attack (Словарная атака)**

Вместо перебора всех символов — перебор слов из словаря (10 000–1 000 000 слов). Работает, потому что люди используют словарные слова.

**Пример:** "sunshine" — в словарной атаке это 1 попытка, а не 95^8 комбинаций.

**Метод 3: Rainbow Tables (Радужные таблицы)**

Заранее вычисленные хэши для миллиардов паролей. Если злоумышленник украл базу хэшей — он просто сравнивает хэш с таблицей.

**Метод 4: Социальная инженерия**

Злоумышленник не взламывает — он "выманивает" пароль.
- Фишинг: поддельная страница входа
- Предлоги: "я из техподдержки, ваш аккаунт взломан, скажите пароль"
- Подглядывание (shoulder surfing)

### 1.3. Как создать пароль, который не взломают

**Правило 1: Длина — это король**
Пароль из 16 символов в 10^18 раз надёжнее пароля из 8 символов.

**Правило 2: Не используй словарные слова**
Даже "iloveyou" — это 1 запись в словаре.

**Правило 3: Используй все типы символов**
Заглавные + строчные + цифры + спецсимволы.

**Правило 4: Один пароль — один сервис**
Утечка на одном сайте = компрометация всех аккаунтов.

**Техника: Passphrase (Фраза-пароль)**
Вместо "суперсложногопароля" — фраза:
`ЯКуп!лНовыйN0utbukВ2025Году!`
- Легко запомнить
- 30+ символов
- Содержит все типы символов
- Не является словарным словом

### 1.4. Менеджеры паролей

**Как работают:**
- Один мастер-пароль (очень сложный, ты помнишь только его)
- Все остальные пароли хранятся в зашифрованном хранилище
- Менеджер сам генерирует случайные пароли (16+ символов)
- Автозаполнение на сайтах

**Примеры:** KeePassXC (бесплатный, локальный), Bitwarden, 1Password

**Важно:** Мастер-пароль должен быть ДЕЙСТВИТЕЛЬНО надёжным — это ключ от всех дверей.

### 1.5. Двухфакторная аутентификация (2FA/MFA)

**Что такое факторы аутентификации:**
1. **Что ты знаешь** (пароль, PIN-код)
2. **Что у тебя есть** (телефон, аппаратный ключ, карта)
3. **Кто ты** (биометрия: отпечаток, лицо, голос)

Один фактор — ненадёжно. Два фактора — в 100+ раз безопаснее.

**Типы 2FA:**
- **SMS-коды** — удобно, но небезопасно (SIM-swap, перехват SS7)
- **TOTP (Time-based One-Time Password)** — Google Authenticator, Authy — надёжно
- **Аппаратные ключи** — YubiKey, Titan — очень надёжно
- **Push-уведомления** — удобно, безопасно
- **Биометрия** — удобно, но нельзя сменить

### Задание 1. Оценка стойкости паролей вручную (10 минут)

**Инструкция:** Ученик получает список паролей. Для каждого нужно:
1. Посчитать количество комбинаций (вычислить вручную или на калькуляторе)
2. Оценить время взлома при скорости 10^9 паролей/сек
3. Определить, уязвим ли пароль к словарной атаке
4. Выставить оценку: 1-10

**Список паролей:**
1. `12345` (только цифры, 5 символов)
2. `kitten` (строчные буквы, 6 символов)
3. `Password123` (заглавные + строчные + цифры, 11 символов — но слово + цифры)
4. `gH9#mK2$` (8 символов, все типы)
5. `MyD0gL0vesT0Run!nTheP@rk2025` (passphrase, 31 символ)
6. `!@#$%^&*` (спецсимволы, 8 символов — но их всего 8)
7. `родина` (русские буквы — 6 символов, но словарное слово)
8. `a1b2c3d4e5f6g7h8` (чередование буква-цифра, 16 символов)

**Разбор с преподавателем:**

1. `12345`: 10^5 = 100 000 комбинаций. Время: 0.0001 сек. Оценка: 1/10. Уязвим — словарная атака.
2. `kitten`: 26^6 = 308 млн. Время: 0.3 сек. Оценка: 1/10. Уязвим — словарная атака (слово из словаря).
3. `Password123`: ~62^11 = 5.2 × 10^19. Время: 1.5 млрд лет при переборе. НО: это словарное слово + цифры. Словарная атака взломает за ~1000 попыток. Оценка: 2/10.
4. `gH9#mK2$`: 95^8 = 6.6 × 10^15. Время: ~76 дней при 10^9/сек. Не словарное. Оценка: 6/10.
5. `MyD0gL0vesT0Run!nTheP@rk2025`: 95^31 ≈ 10^61. Время: космологическое. Passphrase. Оценка: 10/10.
6. `!@#$%^&*`: 33^8 (только спецсимволы = 33 символа) ≈ 1.4 × 10^12. Время: 23 минуты. Оценка: 3/10.
7. `родина`: 33^6 = 1.3 млрд. Время: 1.3 сек. Словарное слово. Оценка: 1/10.
8. `a1b2c3d4e5f6g7h8`: 62^16 ≈ 4.8 × 10^28. Время: космологическое. НО: есть паттерн — злоумышленник может сократить алфавит. Оценка: 7/10.

### Задание 2. Написание учебного брутфорсера на Python (20 минут)

**Инструкция:** Ученик пишет скрипт, который перебирает пароль по словарю (не для атаки на реальные аккаунты — учебный пример). Цель — понять скорость перебора.

**Шаг 1. Пишем базовый скрипт:**

```python
import hashlib
import time

# Хэш целевого пароля (пароль: "hello")
target_hash = hashlib.md5("hello".encode()).hexdigest()
print(f"Целевой хэш: {target_hash}")

# Загружаем словарь (создадим сами)
words = ["apple", "banana", "hello", "password", "sunshine", "qwerty", 
         "123456", "admin", "welcome", "monkey", "dragon", "master"]

start = time.time()
found = None
attempts = 0

for word in words:
    attempts += 1
    word_hash = hashlib.md5(word.encode()).hexdigest()
    if word_hash == target_hash:
        found = word
        break

end = time.time()

if found:
    print(f"[+] Пароль найден: {found}")
    print(f"[+] Попыток: {attempts}")
    print(f"[+] Время: {end - start:.4f} сек")
else:
    print("[-] Пароль не найден в словаре")
```

**Шаг 2. Генерируем большой словарь (100 000+ слов)**

**Шаг 3. Измеряем скорость перебора**

```python
import hashlib
import time
import string
import itertools

target_hash = hashlib.md5("xzya".encode()).hexdigest()  # пароль из 4 букв
start = time.time()
attempts = 0

for combo in itertools.product(string.ascii_lowercase, repeat=4):
    password = "".join(combo)
    attempts += 1
    if hashlib.md5(password.encode()).hexdigest() == target_hash:
        end = time.time()
        print(f"[+] Пароль: {password}")
        print(f"[+] Попыток: {attempts}")
        print(f"[+] Время: {end - start:.2f} сек")
        print(f"[+] Скорость: {attempts / (end - start):.0f} паролей/сек")
        break
```
