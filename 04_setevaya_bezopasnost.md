# Урок 4. Сетевая безопасность

## Методика

**Цель урока:** Сформировать у ученика понимание сетевых угроз и методов защиты в сети.

**Задачи:**
- Разобрать базовую модель сети и протоколы
- Показать, как выглядит сетевой трафик и как его перехватывают
- Научить пользоваться Wireshark для анализа трафика
- Разобрать защиту в общественных Wi-Fi
- Познакомить с VPN и фаерволом

**Тип урока:** Индивидуальный онлайн

**Оборудование:** Компьютер, Wireshark (установленный заранее), доступ к командной строке, браузер

**Критерии успеха:**
- Ученик объясняет разницу между TCP и UDP
- Ученик перехватывает и анализирует пакет в Wireshark
- Ученик настраивает правило фаервола
- Ученик объясняет, как работает VPN

## 1. Теория (15 минут)

### 1.1. Как работает сеть: модель OSI (упрощённо)

Для понимания безопасности достаточно 4 уровней (из 7):

| Уровень | Пример | Что происходит |
|---------|--------|---------------|
| **4. Прикладной** | HTTP, HTTPS, DNS, FTP | То, что видит пользователь |
| **3. Транспортный** | TCP, UDP | Как передаются данные (с контролем/без) |
| **2. Сетевой** | IP | Куда отправляются данные (IP-адреса) |
| **1. Канальный** | Ethernet, Wi-Fi | Физическая передача |

**Важно для безопасности:** Если злоумышленник находится на одном уровне с вами или ниже, он может перехватывать трафик.

### 1.2. TCP vs UDP: в чём разница для безопасности

| Параметр | TCP | UDP |
|---------|-----|-----|
| Соединение | Устанавливает (3-way handshake) | Не устанавливает |
| Надёжность | Гарантирует доставку | Не гарантирует |
| Скорость | Медленнее | Быстрее |
| Применение | HTTP, Email, FTP | Видео, DNS, VoIP |
| Безопасность | Легче отследить (есть порядок) | Сложнее (нет порядка) |

### 1.3. Порт — номер "двери" на компьютере

- Каждое приложение открывает свой порт
- Стандартные порты: 80 (HTTP), 443 (HTTPS), 22 (SSH), 25 (SMTP)
- Закрытый порт — атакующий не может подключиться
- Открытый порт — потенциальная уязвимость

**Проверка портов:** `netstat -an` показывает все открытые порты на компьютере.

### 1.4. Угрозы в сети

**1. Перехват трафика (Sniffing)**
- В общественном Wi-Fi любой может увидеть ваш трафик
- Без HTTPS пароли передаются открытым текстом
- Инструменты: Wireshark, tcpdump

**2. Man-in-the-Middle (MitM)**
- Злоумышленник встраивается между вами и сервером
- Вы думаете, что общаетесь с сервером — а на самом деле с атакующим
- Атака на незащищённый Wi-Fi, подмена DNS

**3. ARP-spoofing**
- В локальной сети злоумышленник подменяет MAC-адрес шлюза
- Весь трафик проходит через атакующего
- Работает внутри одной сети (например, кафе)

**4. DoS/DDoS**
- Перегрузка сервера запросами
- Отказ в обслуживании (доступность нарушена)

### 1.5. Средства защиты

**VPN (Virtual Private Network):**
- Шифрует весь трафик от устройства до сервера VPN
- Даже в открытом Wi-Fi трафик зашифрован
- Скрывает IP-адрес
- **Важно:** VPN не делает анонимным (логгирование трафика), только шифрует

**Фаервол (Firewall):**
- Фильтрует входящий и исходящий трафик
- Может блокировать порты, IP-адреса, протоколы
- Windows: Windows Defender Firewall
- Правило: "Запрещено всё, что не разрешено"

**HTTPS:**
- Шифрует трафик между браузером и сервером
- Использует TLS/SSL
- Замок в адресной строке — признак HTTPS
- **Но:** DNS-запросы могут быть незашифрованы

## 2. Практика (60 минут)

### Задание 1. Разведка сети командами (10 минут)

**Инструкция:** Ученик выполняет команды в терминале (CMD или PowerShell) и анализирует вывод.

**Команда 1: `ipconfig`**
```
ipconfig /all
```
Найти:
- Свой IP-адрес
- MAC-адрес сетевой карты
- IP-адрес шлюза (gateway)
- DNS-сервер

**Команда 2: `ping`**
```
ping google.com -n 10
ping 8.8.8.8 -n 10
```
- Какая задержка (ms)?
- Есть ли потеря пакетов?
- Сравнить пинг по домену и по IP

**Команда 3: `tracert` / `tracert`**
```
tracert google.com
```
- Сколько "хопов" до google.com?
- На каких узлах максимальная задержка?
- Какие страны/города проходят пакеты?

**Команда 4: `nslookup`**
```
nslookup google.com
```
- Какой IP у google.com?
- Какой DNS-сервер ответил?

**Команда 5: `netstat`**
```
netstat -an | findstr LISTEN
```
- Какие порты открыты на компьютере?
- Какие приложения их слушают?

**Вопросы ученику:**
1. Сколько устройств между тобой и google.com?
2. Какой из твоих портов открыт? Это безопасно?
3. Что увидит злоумышленник в той же Wi-Fi-сети?

### Задание 2. Работа с Wireshark (20 минут)

**Инструкция:** Ученик открывает Wireshark и выполняет серию захватов трафика.

**Шаг 1. Первый захват: ICMP (ping)**

1. Открыть Wireshark, выбрать сетевой интерфейс (обычно Ethernet или Wi-Fi)
2. В строке фильтра ввести: `icmp`
3. В отдельном терминале запустить: `ping 8.8.8.8 -n 5`
4. Остановить захват в Wireshark

**Анализ:**
- Найти пакеты с типом Echo Request и Echo Reply
- Посмотреть IP-адреса источника и назначения
- Открыть пакет и разобрать заголовки: Ethernet → IP → ICMP
- Посмотреть время между запросом и ответом (RTT)

**Шаг 2. Захват HTTP (не HTTPS!)**

1. Открыть сайт с HTTP (например, `http://httpbin.org/get`)
   *Внимание: большинство сайтов используют HTTPS. Преподаватель может поднять тестовый HTTP-сервер или использовать `http://example.com`*
2. Фильтр Wireshark: `http`
3. Наблюдать GET-запрос в открытом виде
4. Увидеть заголовки: User-Agent, Host, Accept

**Демонстрация:** Показать, что в HTTP пароли и данные передаются открытым текстом.

**Шаг 3. Захват HTTPS**

1. Открыть `https://google.com`
2. Фильтр: `tls` или `ssl`
3. Показать, что содержимое зашифровано
4. Видно только IP-адреса и порты, но не содержимое

**Сравнение:**
- HTTP: видно всё содержимое запроса
- HTTPS: видно только "что-то передаётся", но не что именно

**Шаг 4. DNS-запросы**

1. Фильтр: `dns`
2. В браузере открыть любой новый сайт
3. Найти DNS-запрос и ответ
4. Обратить внимание: DNS обычно НЕ шифруется (хотя есть DNS-over-HTTPS)

**Демонстрация:** Показать, что даже при HTTPS DNS-запросы видны — злоумышленник знает, какие сайты вы посещаете.

**Вопросы ученику:**
1. Какой протокол использует твой браузер для google.com?
2. Что видит администратор сети о твоём трафике?
3. Что защищает HTTPS и что НЕ защищает?
4. Если злоумышленник в той же Wi-Fi, что он сможет увидеть?

### Задание 3. Настройка правила фаервола (10 минут)

**Инструкция:** Ученик создаёт правило в Windows Firewall, которое блокирует определённое приложение.

**Шаг 1. Создание правила:**

1. Открыть `wf.msc` (Windows Defender Firewall with Advanced Security)
2. Создать новое правило → Правило для порта
3. Блокировать TCP порт 80 (HTTP)
4. Применить

**Проверка:** Открыть браузер и убедиться, что HTTP-сайты не открываются, HTTPS — работают.

**Шаг 2. Создание правила для программы:**

1. Создать правило, которое блокирует определённую программу (например, браузер Chrome) от выхода в интернет
2. Проверить, что браузер блокирован

**Шаг 3. Разбор:**

1. Какие порты должны быть открыты для работы браузера? (80, 443)
2. Какие порты должны быть открыты для игр? (зависит от игры)
3. Почему не стоит открывать порт 3389 (RDP) наружу?

### Задание 4. MITM-симуляция (10 минут)

**Инструкция:** Преподаватель демонстрирует, как работает Man-in-the-Middle (на тестовом стенде или симуляции).

**Вариант A (если техническая возможность на тестовой сети):**
- Использовать BetterCAP или Ettercap в контролируемой среде
- Показать, как перехватывается HTTP-трафик
- Продемонстрировать, что HTTPS защищает от перехвата (браузер покажет предупреждение)

**Вариант B (если нет возможности):**
- Разбор на схеме (Miro / PDF)

**Схема MITM:**
```
Обычное соединение:
Клиент ———— Сервер
         HTTPS (защищено)

MITM:
Клиент ——— Злоумышленник ——— Сервер
         HTTP           HTTPS
(жертва видит          (сервер отвечает
свой сайт, но           нормально)
все данные             
уходят к 
злоумышленнику)
```

**Демонстрация ARP-spoofing:**

1. Показать, как злоумышленник подменяет ARP-таблицу
2. Объяснить, что в общественном Wi-Fi возможно перехватить трафик любого в той же сети
3. Показать защиту: HTTPS + VPN

**Вопросы ученику:**
1. Почему общественный Wi-Fi опасен?
2. Что защитит в общественном Wi-Fi? (HTTPS + VPN)
3. Как понять, что ты под MITM-атакой? (неожиданный сертификат в браузере)

### Задание 5. Тест "Угрозы в сети" (10 минут)

Ученик должен ответить на вопросы письменно:

1. **Что такое порт в контексте сети?**
2. **В чём разница между TCP и UDP?**
3. **Как работает ARP-spoofing?**
4. **Что защищает VPN? Что НЕ защищает?**
5. **Может ли злоумышленник перехватить HTTPS-трафик в Wi-Fi? Если да, то как?**
6. **Какой порт используется для HTTPS?**
7. **Что такое DNS и почему его запросы — угроза приватности?**
8. **Какую команду использовать, чтобы увидеть путь пакета до сервера?**

**Ключи:**
1. Номер "двери" приложения
2. TCP — с подтверждением, UDP — без
3. Подмена MAC-адреса шлюза → трафик идёт через атакующего
4. Шифрует трафик; не защищает от логгирования, от трекинга cookies
5. Если злоумышленник поставил свой сертификат (в корпоративной сети или при установке ПО)
6. 443
7. Отправляет домен в открытом виде
8. `tracert` (Windows) или `traceroute` (Linux/macOS)

## Дополнительно для преподавателя

- Wireshark может напугать ученика сложностью — показать только базовые функции (захват, фильтр, просмотр пакетов)
- Если Wireshark не устанавливается — использовать tcpdump (если Linux) или Online PCAP-анализатор
- Для демонстрации MITM можно использовать готовые PCAP-файлы с атаками
- Подчеркнуть этическую сторону: перехват чужого трафика без разрешения незаконен
