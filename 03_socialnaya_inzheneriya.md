# Урок 3. Социальная инженерия

## Методика

**Цель урока:** Научить ученика распознавать атаки социальной инженерии и противостоять им.

**Задачи:**
- Разобрать психологические принципы манипуляции
- Изучить 7 основных техник социальной инженерии
- Научить анализировать фишинговые письма и сообщения
- Развить критическое мышление при обработке входящей информации
- Провести ролевую игру "атака — защита"

**Тип урока:** Индивидуальный онлайн

**Оборудование:** Компьютер, браузер, Google Docs, подготовленные PDF с примерами фишинга

**Критерии успеха:**
- Ученик называет 7 техник социальной инженерии
- Ученик находит 5+ признаков фишинга в письме
- Ученик успешно отражает атаку в ролевой игре
- Ученик создаёт памятку по противодействию социнженерии

## 1. Теория (15 минут)

### 1.1. Что такое социальная инженерия?

**Определение:** Социальная инженерия — это метод управления действиями человека без использования технических средств, основанный на использовании слабостей человеческой психики.

Кевин Митник — самый известный социальный инженер. Он говорил: "Люди — самое слабое звено в безопасности. Технологии можно обойти, если обмануть человека."

**Ключевая мысль:** Взломать человека дешевле и легче, чем взломать систему.

### 1.2. Психологические принципы (6 принципов Чалдини)

Роберт Чалдини описал 6 принципов влияния, которые используют социальные инженеры:

| Принцип | Суть | Пример атаки |
|---------|------|-------------|
| **Взаимность** | Мы чувствуем обязанность вернуть услугу | "Я помог вам с настройкой, теперь подпишите этот документ" |
| **Дефицит** | Всё редкое кажется более ценным | "Осталось всего 3 места на курсы — запишитесь сейчас" |
| **Авторитет** | Мы доверяем экспертам | "Звонок из службы безопасности банка" |
| **Последовательность** | Мы стремимся быть последовательными | "Вы уже участвуете в нашем опросе, ответьте ещё на пару вопросов" |
| **Социальное доказательство** | Мы делаем как все | "500 человек уже перешли по этой ссылке" |
| **Симпатия** | Мы доверяем тем, кто нам нравится | Злоумышленник притворяется привлекательным человеком |

### 1.3. Семь техник социальной инженерии

**1. Фишинг (Phishing)**
Массовая рассылка писем, имитирующая легитимные сервисы.
- Пример: "Ваш аккаунт заблокирован, перейдите по ссылке для восстановления"
- **Спектр:** Spear-phishing (целевой), Whaling (атака на руководство), Smishing (SMS), Vishing (голосовой)

**2. Претекстинг (Pretexting)**
Создание вымышленной ситуации (pretext) для получения информации.
- Пример: Звонок от "сотрудника техподдержки", который "обновляет систему"
- Отличительная черта: сложная, продуманная легенда

**3. Бейджинг (Tailgating / Piggybacking)**
Проникновение в закрытое помещение вслед за сотрудником.
- Пример: "Я забыл пропуск, придержите дверь" (с коробкой в руках, с кофе)

**4. Кви про кво (Quid pro quo — "услуга за услугу")**
Злоумышленник предлагает помощь в обмен на доступ.
- Пример: "Я из IT-отдела, у нас обновление антивируса — дайте мне ваш пароль, я установлю"

**5. Обратная психология**
"Вы, конечно, не сможете это сделать..." — провокация для получения доступа.
- Пример: "Вряд ли у вас есть доступ к этой базе" — сотрудник доказывает, что есть

**6. Сбор информации из открытых источников (OSINT)**
Злоумышленник собирает данные из соцсетей, LinkedIn, форумов для создания убедительного претекста.

**7. Фейковые формы и страницы входа**
Поддельные страницы, которые выглядят как реальные сервисы.

### 1.4. Признаки фишингового письма

**Чек-лист "Красные флаги":**
1. **Срочность и давление** ("Немедленно! Ваш аккаунт будет удалён!")
2. **Общее обращение** ("Уважаемый клиент", а не "Уважаемый Иван")
3. **Грамматические ошибки**
4. **Подозрительный адрес отправителя** (`support@g00gle.com`, а не `@google.com`)
5. **Ссылка не соответствует тексту** — навести курсор и посмотреть реальный URL
6. **Просьба ввести личные данные**
7. **Необычные вложения** — `.exe`, `.zip`, `.docm` (с макросами)
8. **Угрозы** ("будет заблокирован", "будут приняты меры")

## 2. Практика (60 минут)

### Задание 1. Анализ фишинговых писем (20 минут)

**Инструкция:** Преподаватель показывает ученику 5 писем (реальные примеры). Ученик должен:
1. Определить, фишинг это или легитимное письмо
2. Обосновать решение (минимум 3 признака)
3. Определить тип атаки (phishing, spear-phishing, whaling)
4. Предложить правильные действия получателя

**Письмо 1:**
```
От: noreply@bank-secure-verify.com
Тема: СРОЧНО! Ваш онлайн-банк заблокирован

Уважаемый клиент!

Наша система обнаружила подозрительную активность в вашем аккаунте.
Для восстановления доступа немедленно перейдите по ссылке:
https://bank-secure-verify.com/verify

Если вы не подтвердите личность в течение 24 часов,
ваш аккаунт будет навсегда заблокирован.

С уважением,
Служба безопасности Банка
```
**Признаки фишинга:** срочность, подозрительный домен, общее обращение, угроза блокировки.

**Письмо 2:**
```
От: maria.ivanova@company.ru
Тема: Отчёт за 2-й квартал

Иван, привет!

Направляю тебе обновлённый отчёт за 2-й квартал.
Посмотри, пожалуйста, особенно страницу 5 — там я обновила цифры.

Нужно твоё утверждение до пятницы.

Спасибо!
Мария Иванова
Финансовый отдел
```
**Не фишинг** (если знакомый отправитель, нормальный контекст). Но если письмо неожиданное от коллеги — может быть компрометацией.

**Письмо 3:**
```
От: admin@goog1e.com
Тема: Обнаружен вход с нового устройства

Здравствуйте!

Ваш аккаунт Google был использован для входа с нового устройства:
IP: 185.220.101.42
Устройство: Samsung Galaxy S25
Страна: Нигерия

Если это не вы — перейдите по ссылке для защиты аккаунта:
http://goog1e.com/security-check

Google Security Team
```
**Признаки фишинга:** подмена `google` на `goog1e`, ссылка `http://` (без `s`), общее обращение, создание паники.

**Письмо 4:**
```
От: support@ozon.ru
Тема: Ваш заказ №48592-12 от 05.06.2025

Иван Петров, здравствуйте!

Ваш заказ №48592-12 находится в пункте выдачи.
Состав заказа:
- Рюкзак городской ZW-320 (1 шт)
- Наушники TWS Pro (1 шт)

Забрать заказ можно до 12.06.2025.

Отследить: https://www.ozon.ru/tracking/48592-12

С уважением,
Ozon
```
**Не фишинг** (если ученик действительно заказывал — но нужно проверить из личного кабинета, а не по ссылке).

**Письмо 5:**
```
От: ceo.director@company-offshore.com
Тема: Конфиденциально

Иван!

Я сейчас на переговорах, не могу говорить.
Нужно срочно перевести 340 000 руб на счёт:
Банк: Альфа-Банк
Счёт: 40817810099990001234
Получатель: ООО "Поставщик"

Деньги нужны до 18:00 для завершения сделки.
Подтверждение перевода пришлёшь мне в Telegram.

Александр Сергеевич
Генеральный директор
```
**Признаки фишинга/мошенничества:** срочность, просьба перевести деньги, нестандартный канал связи (Telegram), подозрительный адрес (не корпоративной почты). Тип: Whaling + Претекстинг.

### Задание 2. OSINT-разведка "Что я знаю о тебе" (15 минут)

**Инструкция:** Ученик собирает информацию о себе (или о вымышленном персонаже, заданном преподавателем) из открытых источников. Цель — показать, насколько легко собрать данные.

**Ученик ищет по имени (вымышленного персонажа):**
```
Алексей Смирнов, 14 лет, Москва, школа №1257
```
1. Найти страницы в соцсетях (VK, TikTok, Telegram)
2. Определить по интересам: любит футбол, играет в CS2, смотрит аниме
3. Найти возможный email (логин: `alex.smirnov2009`, `alex_sm1rnov` и т.д.)
4. Найти фото (проверить через TinEye или Google Images)

**После сбора — ученик заполняет профиль:**

```
Имя: Алексей Смирнов
Возраст: 14 лет
Город: Москва
Школа: №1257
Хобби: Футбол, CS2, аниме
Email: alex.smirnov2009@mail.ru (предположительно)
Никнеймы: alex_sm1rnov, leha_football
Телефон: (найден в утечках?)
Друзья: (из ВК)
Родители: (из подписчиков)
Места: (по геотегам)
```

**Демонстрация:** Показать ученику, какая информация доступна незнакомцу за 15 минут.

**Выводы:**
- Пароль "CS2football2010" угадывается за 1 секунду (личные данные!)
- Геолокация по фото в реальном времени — опасно
- Подписки и друзья — информация о близких

### Задание 3. Ролевая игра "Звонок из банка" (15 минут)

**Инструкция:** Ученик играет роль жертвы. Преподаватель — злоумышленник. Ученик должен:
1. Распознать атаку
2. Не дать запрашиваемую информацию
3. Завершить разговор корректно (положить трубку)

**Сценарий:**

*Преподаватель (злоумышленник) звонит (имитация):*
— Здравствуйте, вас беспокоит служба безопасности Сбербанка. Иван Петров?
— Наша система зафиксировала подозрительный перевод с вашей карты на сумму 34 500 рублей в магазин электроники.
— Для блокировки перевода мне нужно подтвердить вашу личность. Назовите, пожалуйста, ваш код из SMS.

*Ученик должен:*
— Не называть код
— Сказать, что сам перезвонит в банк
— Положить трубку

**Варианты развития:**
- Ученик называет код → "перевод подтверждён, деньги украдены" → разбор ошибки
- Ученик кладёт трубку → правильное действие → похвала

**Дополнительный сценарий (если останется время):**

*Преподаватель (злоумышленник) пишет в Telegram:*
— Привет, это Мария Ивановна (классный руководитель). У меня проблема — я не могу зайти в электронный дневник, а нужно срочно выставить оценки. Мне придёт код на телефон, скажи его мне, пожалуйста.

*Ученик должен:*
— Осознать: учитель не будет писать в Telegram и просить код
— Не давать код
— Перезвонить учителю по настоящему номеру

### Задание 4. Создание чек-листа "Проверь входящее сообщение" (10 минут)

**Инструкция:** Ученик создаёт чек-лист из 10 пунктов для проверки любого подозрительного сообщения. Он должен продумать структуру — это алгоритм действий.

**Пример чек-листа (ученик пишет сам):**

```
ЧЕК-ЛИСТ: Это фишинг?

Шаг 1. Оценка отправителя
□ Адрес email совпадает с доменом сервиса? (не mail@go0gle.com)
□ Я знаю этого отправителя лично?

Шаг 2. Оценка содержания
□ Есть ли срочность или давление?
□ Есть ли грамматические ошибки?
□ Просят ли ввести личные данные?

Шаг 3. Оценка ссылок
□ Реальный URL совпадает с текстом ссылки? (навести курсор)
□ URL использует HTTPS?
□ Домен — официальный домен сервиса?

Шаг 4. Оценка вложений
□ Есть ли вложенные .exe, .zip, .docm, .scr?
□ Ожидал ли я это вложение?

Шаг 5. Принятие решения
□ Есть ЛЮБОЙ красный флаг? → Не отвечать, не переходить, не открывать
□ Есть сомнения? → Связаться с отправителем ДРУГИМ каналом
□ Всё чисто? → Всё равно быть осторожным
```

**Критерии оценки:**
- 10+ пунктов — отлично
- 7-9 пунктов — хорошо
- Менее 7 — нужно доработать

## 3. Подведение итогов (15 минут)

### 3.1. Рефлексия

**Вопросы ученику:**
1. Какой приём социальной инженерии кажется тебе самым опасным? Почему?
2. Вспомни случай из жизни, когда кто-то пытался тобой манипулировать.
3. Как изменится твоё поведение в интернете после этого урока?
4. Почему люди продолжают попадаться на фишинг, даже если знают о нём?

### 3.2. Контрольные вопросы

1. Какие 6 принципов влияния по Чалдини ты запомнил?
2. Чем отличается претекстинг от фишинга?
3. Какие 3 признака фишингового письма самые явные?
4. Почему нельзя называть код из SMS даже "сотруднику банка"?
5. Какая информация из твоих соцсетей помогает злоумышленнику?

### 3.3. Домашнее задание

1. Найти в своём почтовом ящике (или в спаме) 2-3 подозрительных письма. Сделать скриншоты и разобрать по чек-листу.
2. Составить "профиль безопасности" своей страницы ВК/TikTok: что видят незнакомцы? Написать отчёт.
3. Провести "аудит" для родителей: показать им 3 признака фишинга на реальных примерах.
4. Написать пост для соцсетей (друзьям) "5 правил, чтобы не попасться мошенникам".

## Дополнительно для преподавателя

- Если ученик не хочет проверять реальный email — используйте тестовый (например, создайте учебный аккаунт)
- Для ролевой игры: меняйтесь ролями — пусть ученик будет злоумышленником, а преподаватель — жертвой
- Подчеркните, что попасться на социальную инженерию может каждый — это не "глупость", а использование естественных психологических механизмов
- История для мотивации: в 2020 году школьник из Флориды с помощью социальной инженерии получил доступ к системе школьных оценок и изменил 1500+ оценок себе и друзьям
